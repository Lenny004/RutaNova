# Fase 4 — Frontend (pantallas del manual)

## Objetivos

- Implementar todas las interfaces del manual con UX profesional y navegación coherente.
- Integrar cada pantalla con las APIs de la Fase 3.

## Tareas

1. Shell de autenticación y layout de app (nav inferior: Inicio, Gestiones, Chats, Perfil).
2. Login con toggle de visibilidad de contraseña y enlace de recuperación.
3. Flujo recuperar / confirmación / restablecer contraseña.
4. Home: bienvenida, mapa, módulos Gestiones / Chats / Dirección actual.
5. Gestiones: lista con estado visual (Pendiente / Cancelado / En curso / Completada).
6. Detalle de gestión: ID, progreso, indicaciones, emisor/receptor, precauciones, mapa, botón Comenzar.
7. Envío en proceso: origen/destino, Waze deep link, modal QR.
8. Chat: filtros Managers/Consumers, lista y conversación con envío.
9. Perfil: datos, estadísticas, edición básica, cerrar sesión.
10. Estados de carga, vacío y error en cada vista.

## Criterios de aceptación

- [x] Todas las pantallas del índice del manual existen y son usables.
- [x] La UI es responsive (móvil primero) y mantiene identidad RutaNova.
- [x] Flujos críticos funcionan end-to-end con backend real.
- [x] No hay pantallas placeholder vacías sin valor.
